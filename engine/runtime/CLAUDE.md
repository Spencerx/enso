# engine/runtime

The Truffle/GraalVM interpreter for Enso. The **language-side** code (everything
that executes inside the polyglot context). Package root `org.enso.interpreter`.

Read `docs/runtime-guide.md` before making non-trivial changes — it maps entry
points, documents threading, and links the required Graal/Truffle background
reading.

## Key classes

- `EnsoLanguage.java` — The `TruffleLanguage` implementation. Entry point for
  everything.
- `EnsoContext.java` — Per-context state (module registry, builtins,
  distribution paths, thread manager).
- `runtime/builtin/Builtins.java` — Synthesizes the `Standard.Builtins` virtual
  module from annotation-processor output. Changing a builtin touches both the
  Java class and the `@BuiltinMethod` annotations that drive codegen.

## Subpackages

- `node/` — Truffle `Node` classes that make up the AST (callable handling,
  control flow, expressions, scopes, type checks).
- `runtime/` — Non-AST runtime helpers: module system, data (atoms, vectors,
  numbers, text, warnings, hash maps, …), builtins, callables, control
  (exceptions, TCO), scope, state, resource management.
- `service/` — Runtime services exposed over `RuntimeServerInstrument` to the
  language server.
- `caches/` — IR/module cache machinery (persisted compilations for faster
  startup).
- `util/` — Shared helpers.

## Non-obvious conventions

- **`public static build(...)`** on every Truffle node used in an AST.
  DSL-generated nodes delegate to the autogenerated `create(...)`. Nodes never
  meant to appear in ASTs (e.g. builtin-function root nodes) expose
  `asFunction()` instead.
- **`InteropLibrary` only** for language-side dispatch. Never import
  `org.graalvm.polyglot`. Never depend on `truffle-api` from outside this tree.
- **`@CompilationFinal` + `final`** — Truffle does constant folding on these.
  Use them aggressively for fields that are set once during compilation /
  initialization. Pair with `@ExplodeLoop` when a loop bound is
  compilation-final.
- **DSL `@Specialization`** — Read generated code before guessing. Subtle bugs
  hide in specialization-ordering and guard interactions.
- **Safepoints.** Every iteration of a TCO loop and every non-inlined method
  call should poll safepoints. See `runtime/ThreadManager.java`.
- **No `System.out.println`.** Use `TruffleLogger` in language code; the
  embedder configures sinks.

## Language Server integration

The `RuntimeServerInstrument` (under `runtime-instrument-runtime-server/`, not
here) attaches to this runtime. Messages flow over `polyglot-api` types using a
stringly-typed method-name convention — chase those constants to cross the
embedder/language boundary.

## Tests

Ultimately driven from SBT (`runtime-integration-tests/test`). Scalatest + Enso
stdlib tests that exercise the whole compiler + runtime. Don't add Java-side
mocks for runtime behavior — tests should go through a live context.

## JNI native parser

At startup, `EnsoContext` loads `enso_parser` (the JNI `cdylib` from
`lib/rust/parser/jni/`). Make sure the library is on `java.library.path` or
inside the engine distribution under `component/runtime/`.
